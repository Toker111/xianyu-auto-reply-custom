# 当前会话 Markdown 导出与专业 AI 商品上下文

## 当前会话导出

- 入口：`在线聊天`页面，选中一个买家会话后，点击聊天区右上角的`导出 Markdown`。
- 范围：只导出当前打开的单个买家会话。
- 完整性：先使用页面已加载的消息，再按聊天分页游标继续读取更早的消息，直到该会话没有更多历史记录。
- 内容：导出时间、卖家账号显示名、买家显示名、当前商品公开资料、按时间排序的聊天文字和图片链接。
- 文件在浏览器本机生成并下载，不会上传到额外的导出服务。
- 导出失败或分页异常时不会伪装成完整文件，页面会提示重试。

聊天内容本身可能含有买卖双方主动发送的信息。导出文件应按私人资料保存，不要公开分享，也不要提交到 Git。

## AI 如何知道当前商品

生成 AI 建议前，前端读取本地商品目录：

1. 优先使用会话携带的商品 ID 精确匹配；
2. 会话没有商品 ID 时，使用商品标题精确匹配；
3. 匹配成功后，把公开的商品标题、商品描述和商品价格放入本次 AI 请求；
4. 匹配不到时仅使用会话已有的商品标题，不猜测详情。

这一步不会为了补充商品资料重新抓取闲鱼。Cookie、Token、密码、API Key、验证码、订单隐私不会作为商品上下文发送。商品资料和自定义提示词仍会经过本机敏感信息检测，命中时要求人工处理。

## 专业回答规则

AI 建议的系统提示词要求模型：

- 使用模型自身通用/专业知识解释问题，同时以当前商品公开资料为事实依据；
- 专业问题先给明确结论，再解释关键依据、适用条件和注意事项；
- 不堆砌术语，不编造规格、兼容性、库存、优惠、订单状态、售后或发货承诺；
- 信息不足时只问一个最关键的问题；
- 只输出可供人工审核的一条建议回复，不输出分析过程。

AI 建议仍然不会自动发送给买家，必须经过已有的人工确认、修改或忽略流程。
