# AI 建议模式设计（第一阶段）

## 目标与边界

AI 只分析已经由人工批准、可发送给 AI 的聊天内容，并生成一条“建议回复”。建议不会自动发给买家，必须由人工发送、修改后发送、重新生成或忽略。

本模式与现有手动回复、自动回复功能并存，不删除旧功能。账号可在“手动 / AI 建议 / AI 自动”三种模式间选择；第一阶段只启用和完善“AI 建议”。

## 消息进入 AI 前的人工检查

1. 连续到达的买家文本消息合并为一个大边框消息组。
2. 组下方只有一套操作：立即发送给 AI、拒绝发送给 AI、编辑副本后发送给 AI。
3. 默认倒计时 4 秒；新消息加入当前组时，整组倒计时重置为 4 秒。
4. 只有当前打开的会话倒计时。离开会话暂停；再次进入时重置为完整倒计时。
5. 编辑消息组时倒计时停止。编辑期间到达的新消息进入下一组，当前组处理完后再开始下一组。
6. 人工在消息组等待期间直接回复买家时，取消本次自动 AI 分析，并提供“仍然生成建议”。
7. 图片、附件和二维码默认不发给 AI。

## 敏感信息

人工检查是主要防线。本机还会在后端进行确定性检测；检测到下列高风险内容时停止倒计时并阻止自动提交：

- 密码或口令；
- Cookie、Token、Authorization；
- API Key、Secret Key；
- 短信或邮箱验证码。

系统只返回风险类型和所在消息，不把命中的秘密写入 AI 上下文、建议记录或日志。用户必须编辑副本后重新提交，或拒绝该组。

## 上下文与顺序

- 本地 AI 可见历史不设条数上限。
- 每次调用按模型上下文限制发送最近消息；较早内容后续使用本地摘要或检索补充，原始本地历史不删除。
- 消息使用会话内单调递增序号排序，不依赖客户端时间。
- 人工发送成功的卖家消息先在本地等待，在下一次买家触发组中一起供人工确认并同步给 AI。
- AI 建议只有在真正发送给买家后才进入 AI 可见历史。忽略、失败和未发送草稿不进入上下文。
- 拒绝组只保留“不向 AI 提供内容”的占位状态，不保存原文。

## 建议回复交互

生成结果显示在聊天输入框上方，一次只显示一条建议：

- 发送给买家；
- 原地修改后发送；
- 重新生成（可填写一次性补充要求）；
- 忽略。

生成失败时不发送任何内容，只提示人工回复。

## AI 配置

- 全局配置：管理员维护连接配置、默认配置、明确的失败回退顺序和默认回复风格。
- 局部配置：账号可继承全局配置，或覆盖 AI 连接、审核倒计时和回复风格。
- 官方 DeepSeek 使用原生配置入口，固定官方地址，默认模型使用当前官方模型名。
- 中转站和其他服务统一使用 OpenAI 兼容接口，可配置名称、Base URL、模型和 API Key。
- 普通用户只能使用被分配的配置，不能查看或修改 API Key。
- API Key 加密保存；读取接口永不回显，只显示“已配置”。修改时只能用新值替换。
- 测试连接使用固定测试文本，不携带任何闲鱼聊天内容。
- 未明确配置回退链时，失败后绝不静默切换服务商。

## 商品、订单与审计

默认可发送给 AI 的业务信息只有商品标题、公开描述、价格和非敏感订单状态。姓名、电话、地址、支付信息不发送。

建议记录保存状态（已生成、已发送、修改后发送、忽略、失败）、配置、服务商、模型、耗时、输入/输出 Token 和估算费用。记录只保存已经批准给 AI 的内容，不保存被拒绝原文、API Key、Cookie 或 Token。
