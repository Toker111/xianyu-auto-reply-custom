# 账号级人工滑块验证模式

## 目的

当闲鱼账号触发阿里官方风控验证时，允许用户为单个账号选择“人工验证”，避免程序继续自动拖动或同时启动多个验证窗口。

该模式不会绕过平台验证。用户必须在官方验证页面中亲自操作，平台仍有权拒绝当前浏览器环境。

## 开关位置

1. 打开“账号管理”。
2. 找到目标账号，点击“编辑”。
3. 在“自动登录设置”区域找到“人工滑块验证模式”。
4. 勾选并保存表示开启；取消勾选并保存表示关闭。

开关按账号独立保存，新账号和现有账号默认均为关闭。

## 开启后的行为

- 仅对当前账号生效。
- 检测到风控后打开一个可见 Chromium 窗口。
- 不执行程序鼠标移动、页面滚动或自动拖动。
- 禁用远程、真实鼠标、Playwright 自动拖动和 DrissionPage 兜底求解器。
- 用户约有 3 分钟完成验证。
- 同账号沿用浏览器互斥锁，避免同时打开多个验证窗口。
- 只有官方页面离开 punish 状态并产生真实的 `x5sec` Cookie 时才判定成功。
- Cookie 值不会写入新增日志。

## 关闭后的行为

恢复项目原有的验证码处理流程。关闭开关不会删除 Cookie、浏览器资料、数据库数据或日志。

## 使用注意

- 连续失败时先禁用账号并暂停尝试，避免提高平台风控等级。
- 不要清理 Token、Cookie 或 `browser_data` 来反复重试。
- 不要把密码、Cookie、Token 或验证码截图交给 Codex。
- 人工模式仅支持有桌面的 Windows 源码部署。
- 即使是人工操作，平台仍可能拒绝内置自动化 Chromium；此时应停止尝试，使用平台官方支持的登录和验证方式。
